Lesson 2

Storing Data

In Computer Applications

When you do homework on a computer and save your work, you give it a name so you can find it later. This name can be anything you want it to be as long as you follow some rules. The name cannot have a /, \, *, ?, “, :, <, >, and | (on a Windows system). If you accidentally save another file with the same name, you’ll cry (because you just overwrote your homework!).

Similarly, when you take pictures and transfer them to your computer, you can modify the picture names. The same rules apply, and the same overwriting effects can occur.

In Python

Storing data in Python is very much like storing files on your computer. In Python, the data is stored in variables. However, there are more rules that need to be followed. The variable name must:

Start w/ A-Z, a-z
Contain only A-Z, a-z, 0-9, _
Not contain spaces, punctuation, special characters, except _

Programs are written to be read and maintained by others (and you). Clarity is essential. The following are some examples of good variable names because they describe the data they’re storing.
robotSpeed
iceCreamPrice
distanceTraveled
mpg
high_score

The following are some examples of bad variable names:
high score – variable names cannot have spaces
robtSpd – confusing abbreviation
1stPlace – variable names cannot start with a digit
x – this variable name is not descriptive
o – this variable name is not descriptive and can be confused with a zero

To store data in a variable, use an equal sign. This operation is called assignment. To display the stored value, use print. For example:
distanceTraveled = 100
iceCreamPrice = 2.5
print distanceTraveled
print iceCreamPrice

Output:
100
2.5

In the example above, 100 was stored in the variable, distanceTraveled and 2.5 was stored in the variable, iceCreamPrice. Notice that the units are not specified. We don’t know if distanceTraveled is 100 miles or 100 kilometers. Just like in a calculator, the programmer is responsible for keeping track of the units.

Here is another example:
myHighScore = 777777
myRivalScore = 100000
print myHighScore
print myRivalScore

print "My rival played the game again"
myRivalScore = 999999
print myHighScore
print myRivalScore

Output:
777777
100000
My rival played the game again
777777
999999

In this example, note that similar to overwriting a file on a computer, the variable, myRivalScore, was overwritten to 999999. Notice that the value of myHighScore was unchanged because we did not assign it a new value.
