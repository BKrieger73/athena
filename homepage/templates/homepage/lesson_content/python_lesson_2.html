Lesson 2

Storing Data

In Computer Applications

When you do homework on a computer and save your work, you give it a name so you can find it later. This name can be anything you want it to be as long as you follow some rules. The name cannot have a /, \, *, ?, “, :, <, >, and | (on a Windows system). If you accidentally save another file with the same name, you’ll cry (because you just overwrote your homework!).

Similarly, when you take pictures and transfer them to your computer, you can modify the picture names. The same rules apply, and the same overwriting effects can occur.

In Python

Storing data in Python is very much like storing files on your computer. In Python, the data is stored in variables. A variable is a space in memory where you save data. However, there are more rules that need to be followed. Each variable must have a name so that you can identify the data that is stored there. The variable name must also:

start with any letter, a to z (and capital letters are fine too)
contain only letters, numbers, and the underscore symbol, _
not contain spaces, punctuation, or any other special characters

Programs are written to be read and maintained by you, the programmer, or other programmers. Clarity is essential. The following are some examples of good variable names because they describe the data they’re storing.
robotSpeed
iceCreamPrice
distanceTraveled
mpg
high_score

The following are some examples of bad variable names:
high score – variable names cannot have spaces
robtSpd – confusing abbreviation
1stPlace – variable names cannot start with a digit
x – this variable name is not descriptive
o – this variable name is not descriptive and can be confused with a zero

Your First Variable

To store data in a variable, use an equal sign. This operation is called assignment. To display the stored value, use print. For example:
distanceTraveled = 100
iceCreamPrice = 2.5
print distanceTraveled
print iceCreamPrice

Output:
100
2.5

In the example above, in the first line, 100 was stored in the variable in, distanceTraveled. In the second line, 2.5 was stored in the variable, iceCreamPrice. Then, in the third line, the program looks at the data within distanceTraveled and finds that 100 is stored there. Then it goes ahead and runs the print part of the third line and displays 100 in the output. Lastly, the fourth line looks up the data in iceCreamPrice and it is then printed as well. Try this example out in the scratch pad. Note that there are no quotes required for printing variables. Also, notice that the units are not specified. We don’t know if distanceTraveled is 100 miles or 100 kilometers. Just like in a calculator, the programmer is responsible for keeping track of the units.

You vs. Your Rival

Here is another scratch pad example:
myHighScore = 777777
myRivalScore = 100000
print myHighScore
print myRivalScore
print "My rival played the game again"
myRivalScore = 999999
print myHighScore
print myRivalScore

Output:
777777
100000
My rival played the game again
777777
999999

In this example, the first four lines are like the previous example. You store two values in two different variables and print them out. Then, on the 5th line, you print out the string, "My rival played the game again". In the 6th line, your rival got a higher score than you and that value is now stored in myRivalScore. Note that this is similar to overwriting a file on a computer. Then in the 7th and 8th line, the variables are printed out again. Notice that the value of myHighScore was unchanged because we did not assign it a new value.
